# 📅 Генератор расписания интенсивов «Прознание + Coddy»

Веб-приложение для построения расписания интенсивов:
- 3 параллельные локации (Пожарный 19 / Пожарный 3Б / Спутник)
- группировка по месяцам (июнь / июль / август)
- маркеры смен лагеря (МНО / Творческая / Смешанная / IT) — вписываются вручную
- гибкое время каждого занятия (11-13, 14-16, 10:30-13 для длинных)
- бейдж «4 дня / 5 дней» для коротких интенсивов
- библиотека тем в Google Sheets
- встроенный ИИ для подбора тем, расширения описаний и заполнения пустых слотов
- публикация ссылки для родителей/команды (view.html)

---

## Архитектура (та же что у лагерного генератора)

```
Браузер (admin/view)
    ↓ fetch
GitHub Pages (статичные index.html + view.html)
    ↓ fetch
Google Apps Script Web App (Code.gs)
    ↓ SpreadsheetApp / UrlFetchApp
Google Sheets / Anthropic API
```

**Важное отличие** от лагерного: в одном Apps Script могут жить и лагеря, и интенсивы — каждый запрос помечен `project=intensive` или `project=camp`. Для интенсивов используются отдельные листы (`Активности_интенсивы`, `Опубликованные_интенсивы`, и т.д.).

## Как развернуть с нуля

### 1. Google Sheets

Можно использовать **тот же документ**, что и для лагеря — листы интенсивов будут жить рядом. Или создать новый.

1. Откройте Google Sheets, скопируйте ID из URL: `docs.google.com/spreadsheets/d/<<<SHEET_ID>>>/edit`
2. Создавать листы вручную **не нужно** — скрипт создаст их сам при первом обращении. Заголовки появятся автоматически.

Если хотите создать заранее — нужны такие листы с такими заголовками:

**Активности_интенсивы**:
| Статус | Шаблон занятия | Примеры тем | Педагог | Направление | Локация / кабинет | Возраст | Длительность (мин) | Формат | Описание |

**Реквизиты_интенсивы** (для PDF/брендинга):
| Ключ | Значение |

Остальные листы (`Опубликованные_интенсивы`, `Черновики_интенсивы`, `Сессии_интенсивы`) трогать руками не нужно.

### 2. Apps Script

1. В Google Sheets → **Расширения → Apps Script**
2. Скопируйте содержимое `Code.gs` из этого репо в редактор
3. В строке `const SHEET_ID = '...'` подставьте ваш ID
4. Project Settings (⚙️) → Script Properties → Add property:
   - `ANTHROPIC_API_KEY` = ваш ключ от console.anthropic.com (нужен для работы ИИ)
5. Save (Ctrl+S)
6. Запустите функцию `_testInitSheets_intensive` — она создаст все нужные листы
7. **Deploy → New deployment → Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
8. Скопируйте **Web app URL**

### 3. GitHub Pages

1. Создайте репо `intensive-generator` (или добавьте файлы рядом с лагерным генератором)
2. В файлах `index.html` и `view.html` найдите строку
   ```js
   const LIBRARY_URL = 'https://script.google.com/macros/s/REPLACE_WITH_YOUR_DEPLOYMENT_URL/exec';
   ```
   и подставьте ваш Web app URL
3. Settings → Pages → Source: `main` branch, root
4. URL вида `https://username.github.io/intensive-generator/`

### 4. Тест

- Откройте `https://username.github.io/intensive-generator/`
- Должна загрузиться сетка с 13 неделями
- В правом нижнем углу — тост «Библиотека: N тем» (или «недоступна» если ещё ничего не добавлено)
- Если падает — F12 → Console, скорее всего неправильный `LIBRARY_URL`

## Часть 1 — Для пользователя (как составить расписание)

### Главный workflow за 5 минут

1. **Название**: впишите в шапке — «Интенсивы лето 2026», возраст, дефолтную стоимость
2. **Локации**: 3 жёлтых заголовка сверху — клик правит название
3. **Маркеры смен лагеря** (серые полоски между неделями) — вписываете вручную из своего планирования (МНО / Творческая / Смешанная / IT). Это для вашей ориентировки
4. **Даты недель** — клик по зелёному бейджу даты, правите вручную («1 - 5 июня»)
5. **Слоты** — каждая ячейка имеет 2 строки по умолчанию (11-13 и 14-16). Кликайте и заполняйте: время · тема · педагог. ⋯ — открывает развёрнутую модалку со стоимостью и описанием
6. **Длинные интенсивы (4-5 дней)** — кнопка «+ длит.» → подпись «4 дня», и в слоте смените время на «10:30-13»
7. **Опубликовать** — кнопка наверху справа → ссылка готова

### ИИ-помощники (требуют настроенного API-ключа в Apps Script)

- **⚡ ИИ: заполнить пустые слоты** — заполняет все пустые слоты разом, учитывая контекст недели и маркер смены
- **💡 ИИ: подобрать тему** — для одного слота, с учётом библиотеки и того что уже стоит на этой неделе
- **✨ ИИ: расширить описание** — внутри модалки слота, генерирует текст для родителей

### Действия со слотом (клик ⋯)

- редактировать время / тему / педагога / стоимость / описание вручную
- 💡 подобрать тему через ИИ
- ✨ расширить описание через ИИ
- 📚 выбрать тему из библиотеки

### Работа с публикациями

Кнопка **🌐 опубликовать** открывает панель:
- **Текущая ссылка** для отправки
- **Все публикации** в архиве: 📋 копировать, ↗ открыть, 📥 загрузить обратно в админку, 🗑 удалить
- **🆕 новая публикация** — отвязать текущий state, чтобы следующий клик создал новую запись

Индикатор-кружок возле кнопки:
- серый — публикаций нет
- жёлтый — есть несохранённые правки

### Черновики

Кнопка **📁 черновики** — сохраните промежуточное состояние перед публикацией. Несколько черновиков можно держать параллельно.

### Бэкапы

- **⬇ экспорт JSON** — скачать всё текущее состояние файлом (на всякий случай, если что-то пошло не так)
- **⬆ импорт JSON** — восстановить из файла

### Если что-то пошло не так

- **Случайно нажала** «🧹 очистить все слоты» / перегенерировала / удалила — жмите **↶ отменить** или **Ctrl+Z** (до 20 шагов назад)
- **Загрузила старую публикацию через 📥, потерялся текущий state** — жмите **↶ отменить** (вернёт state до загрузки)

## Часть 2 — Для оператора-помощника (библиотека тем)

Библиотека интенсивов в Google Sheets, лист **«Активности_интенсивы»**.

### Что писать в колонки

| Колонка | Пример |
|---|---|
| **Статус** | `утверждено` (только такие подгружаются в HTML; `черновик` — не подгружается) |
| **Шаблон занятия** | `Взросление.Девочки`, `Roblox`, `ИИ в учёбе`, `Стикеры` |
| **Примеры тем** | через запятую |
| **Педагог** | `Власова`, `Пашкевич`, `Кургузикова`, `Козырев`, `Парджанадзе`, `Печковская`, `Злата` |
| **Направление** | `Soft skills`, `IT`, `Творчество`, `МНО` |
| **Локация / кабинет** | если важно |
| **Возраст** | `6–11 лет` |
| **Длительность (мин)** | обычно `120` |
| **Формат** | `интенсив` |
| **Описание** | что ребёнок узнаёт за неделю — для ИИ и для родителей |

### ❌ Что НЕЛЬЗЯ делать вручную

1. Менять имя педагога правкой ячейки — в опубликованных сетках останется старое имя. (Каскадное переименование пока не реализовано — но можно через 📥 загрузить публикацию обратно, поправить и обновить)
2. Менять заголовки колонок в первой строке — скрипт ищет по точным именам
3. Удалять колонку «Статус»

### ✅ Что МОЖНО

- Добавлять новые темы — просто новая строка в конец со статусом `утверждено`
- Редактировать описания — повлияет только на новые слоты, на старые публикации — нет
- Делать темы временно невидимыми, поменяв статус на `черновик`

## Часть 3 — Для разработчика

### Стек

- **Frontend**: статические `index.html` + `view.html` на GitHub Pages, без билда. Весь JS внутри HTML
- **Backend**: Google Apps Script (`Code.gs`) развёрнут как Web App
- **БД**: Google Sheets (один документ, несколько листов с префиксом `_интенсивы`)
- **AI**: Anthropic Claude (через прокси в Apps Script, ключ в Script Properties)

### Структура state (что лежит в localStorage / в публикации)

```javascript
{
  title: "Интенсивы лето 2026",
  ages: "6–11 лет",
  defaultCost: 40,
  locations: ["Пожарный 19", "Пожарный 3Б (Детали)", "Спутник"],
  publishId: "i_abc123def456",  // null если ещё не публиковали
  _draftId: null,
  weeks: [
    {
      id: "w_xxxxxx",
      month: "июнь",            // для группировки в шапки
      marker: "Смешанная",       // МНО / Творческая / IT / Смешанная
      locations: [               // ровно 3, по индексу = state.locations
        {
          dateRange: "1 - 5 июня",
          multiDayLabel: "",     // или "4 дня" / "5 дней"
          slots: [
            { time: "11-13", theme: "...", teacher: "...", cost: "40", description: "...", template: "...", color: "green", locked: false },
            { time: "14-16", ... }
          ]
        },
        // ... ещё 2 локации
      ]
    },
    // ... ~13 недель
  ]
}
```

### Apps Script — actions

**GET** (`?action=...&project=intensive`):
- `all` / `library` / `details` — библиотека и реквизиты
- `listPublished`, `getPublished`, `listDrafts`, `loadDraft`
- `listSessions` — кто сейчас редактирует

**POST** (JSON body с `action` и `project`):
- `claude` — прокси к Anthropic API
- `publish` / `unpublish`
- `saveDraft` / `deleteDraft`
- `heartbeat` / `endSession`

### Один Apps Script для лагеря и интенсивов

Если у вас уже работает лагерный генератор:
- Можно скопировать функции из этого `Code.gs` в существующий — они работают через параметр `project`. Но **проще** держать два деплоя: лагерный остаётся как есть, для интенсивов делаете отдельный Web app URL
- Или добавить новый Apps Script в ту же таблицу — тогда два URL, не пересекаются

### Файлы

- `index.html` (~1100 строк) — админка
- `view.html` (~330 строк) — страница просмотра
- `Code.gs` (~430 строк) — серверная логика

### Локальная разработка

```bash
git clone <repo>
cd intensive-generator
python3 -m http.server 8000
# открываете http://localhost:8000/
```

LocalStorage сохраняет состояние между перезагрузками — даже если LIBRARY_URL фейковый, можно работать локально и потом импортировать через JSON.

### Обновление кода (после правок)

**HTML на GitHub Pages**:
1. Залить новый `index.html` или `view.html` через web-интерфейс GitHub
2. Подождать 30–60 сек на пересборку Pages
3. В браузере: `Ctrl+Shift+R` (сброс кэша)

**Apps Script**:
1. Save (Ctrl+S)
2. **Deploy → Manage deployments → ✏️ Edit → Version: New version → Deploy**
3. URL остаётся тот же

### Известные ограничения

- **CORS**: Apps Script не возвращает CORS-заголовки → POST идёт в `mode: 'no-cors'`, ответ не читаем напрямую. Для важных операций (публикация) делаем verify-loop через GET
- **setValues и длинные строки**: `setValues()` обрезает строки длиннее 50000 символов. Поэтому при обновлении публикации делаем `deleteRow + appendRow`, а не `setValues`
- **Caching**: GitHub Pages кэширует через CDN до 10 минут — после правок жмите Ctrl+Shift+R
- **Anthropic rate limits**: ~30k input tokens/min. Заполнение всех 78 слотов разом может упереться → ИИ это делает в одном запросе с компактным промптом

### Что **не** реализовано (возможные направления)

- Каскадное переименование педагога/шаблона (как в лагерном) — пока нет
- Расширение библиотеки через ИИ прямо из админки — нет, только через Sheets
- PDF тизер для родителей — пока нет (можно скопировать логику из лагерного)
- Аутентификация — нет, доступ только по URL
- Mobile-friendly режим — частично, но 3 колонки превращаются в 1 (вертикально)
